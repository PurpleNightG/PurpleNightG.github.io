# 🚀 GitHub Pages 部署步骤

## ⚠️ 重要提示
请在**之前初始化 Git 的终端**中执行以下命令（确保有 Git 命令）

---

## 📝 执行以下命令

### 1. 添加所有项目文件到 Git
```bash
git add .
```

### 2. 提交更改
```bash
git commit -m "部署到 GitHub Pages - 初始版本"
```

### 3. 推送到 GitHub（强制推送，覆盖之前的 README）
```bash
git push -f origin main
```

> 💡 使用 `-f` 是因为远程仓库有一个初始的 README.md，需要覆盖它

---

## 🎯 推送成功后的配置

### 1. 访问 GitHub 仓库
https://github.com/PurpleNightG/PurpleNightG.github.io

### 2. 启用 GitHub Pages
1. 点击仓库的 **Settings** 标签
2. 在左侧菜单找到 **Pages**
3. 在 "Build and deployment" 部分
4. **Source** 选择 **GitHub Actions**（不是 Deploy from a branch）

### 3. 查看部署进度
1. 点击仓库的 **Actions** 标签
2. 你会看到 "Deploy to GitHub Pages" 工作流正在运行
3. 等待绿色的 ✅ 表示部署成功

### 4. 访问你的网站
部署成功后，访问：
**https://purplenightg.github.io**

## 🐛 常见问题

### Q: 推送时提示 "failed to push some refs"
A: 使用 `git push -f origin main` 强制推送

### Q: GitHub Pages 显示 404
A: 
1. 确认 Settings > Pages 中选择了 "GitHub Actions"
2. 检查 Actions 标签页，确认工作流运行成功
3. 等待 5-10 分钟让 DNS 生效

### Q: 网站能访问但登录失败
A: 检查浏览器控制台错误，确认：
1. `.env.production` 中的 API URL 正确
2. 后端 Vercel 已配置环境变量并重新部署
